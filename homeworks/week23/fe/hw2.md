## 為什麼我們需要 Redux？

當 React 的資料結構變複雜的時候，要一層一層傳遞需要的資料會變得很麻煩，中間經過的 component 有可能不需要用到，使用 Redux 除了可以更方便，也可以優化效能。

## Redux 是什麼？

Redux 是一種儲存狀態的容器，是一種設計模式，不只能使用在 React 上。

## Single Page Application 是什麼？有哪些頁面一定要用這個架構去設計嗎？

單頁式應用，就像我們使用手機 APP 那樣，換頁時不會出現空白的等待期，頁面上的資料不會因為換頁而遺失，我們只更新需要更新的部分，保留想留在原地的資料，增進使用者體驗。

可以設計的範例如：音樂播放網站，當我們切換去瀏覽其他列表時，音樂還是能繼續播放，另外還有像直播（Twitch）、影片（Youtube）等等。

## Redux 如何解決非同步（例如說 call API 拿資料）的問題

利用 middleware，像是 redux-thunk 或 redux-promise，透過 dispatch() 串接，最後回到同步的 Redux 資料流。
